# Codex AI Presence Nightly

This repository contains prerelease snapshots generated from the development lab. It is a safe place to try changes that have passed the automated, no-model E2E gate but have not yet been promoted to the clean installable skill repository.

Use the stable skill repository for normal installation:

<https://github.com/walkingIssue/Codex-AI-presence>

Nightly artifacts are generated by GitHub Actions from `walkingIssue/Codex-AI-presence-lab`. They are not the source of truth and may change or disappear without compatibility guarantees.

## Repository roles

- [Lab](https://github.com/walkingIssue/Codex-AI-presence-lab): source, roadmap, experiments, E2E gate, and promotion tooling.
- [Release](https://github.com/walkingIssue/Codex-AI-presence): clean installable skill.
- Nightly: prerelease packages produced from the lab.

The nightly workflow runs on a schedule and can also be started manually from the Actions tab. A manually requested run can publish a GitHub prerelease containing the validated skill archive.

## Recommended client path

For the current nightly app-server streaming bridge, use the Codex CLI/TUI
through the project-local launcher. From the root of a project with the skill
installed:

```powershell
& .\.codex-voice\launch_codex.ps1
```

This starts a separate Codex TUI session through the local WebSocket bridge.
Visible response deltas can then reach Kokoro and the Strand Orb as they
arrive. To resume a known session through the bridge, pass its session ID:

```powershell
& .\.codex-voice\launch_codex.ps1 --client-command '"{codex}" resume --remote {remote} <SESSION_ID> --no-alt-screen'
```

The packaged Codex Desktop GUI is not currently routed through this bridge.
Desktop starts its own private stdio app-server child, so the existing GUI
window will not automatically gain streamed voice or Orb interception. Use
the TUI launcher for nightly bridge testing; GUI attachment remains a future
host-integration track.
